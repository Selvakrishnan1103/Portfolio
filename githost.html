Launching Your Site: GitHub Pages Made Simple


Adding a Remote Origin


First, you need to add a remote origin to your repository. This allows you to push your local changes to the remote GitHub repository. Use the following command:

git remote add origin https://github.com/yourusername/your-repository.git

Pushing Files to the Repository


Once you've added the remote origin, push your files to the repository:

git add .
git commit -m "Initial commit"
git push -u origin main

Setting Up GitHub Pages


To host your site using GitHub Pages, follow these steps:

  1. Navigate to the repository on GitHub.
  2. Go to Settings.
  3. Scroll down to the Pages section.
  4. Select the main branch and set the root directory.
  5. Click Save.

Enabling HTTPS


GitHub Pages provides a free HTTPS certificate for your site. To enable it:

  1. Go to the Pages section in your repositoryâ€™s settings.
  2. Under Custom domain, check the box for Enforce HTTPS.
  3. Check your page by visiting https://yourusername.github.io/yourrepositoryname.

Configuring DNS for a Custom Domain


If you have a custom domain, you can configure your DNS to point to GitHub's servers:

  1. Go to your domain registrar's website.
  2. Add or modify the A records to point to the following GitHub IP addresses:
    • 185.199.108.153
    • 185.199.109.153
    • 185.199.110.153
    • 185.199.111.153

Creating a CNAME File


To ensure your custom domain works with GitHub Pages, create a CNAME file in the root directory of your repository:

echo "yourdomain.com" > CNAME

Conclusion


By following these steps, you can successfully host your webpage using Git and GitHub Pages, with or without a custom domain. GitHub provides a reliable and straightforward way to make your site live, complete with HTTPS support and easy DNS configuration.
